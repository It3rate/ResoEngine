# Conceptual Schemas

This folder is for reusable symbolic schemas that sit between raw Core 2 structure and applied vocabularies such as English words.

The goal is not to hard-code "language" into the symbolic layer.
The goal is to define a small abstract set of reusable relation families that can later be invoked by:
- graph traversal
- dynamic equations and mathematical flow
- flow control
- circuits
- ornamental and spatial patterns
- sentence and story structure
- applied language vocabularies
- image-schema and localist-style inventories

## Why This Layer Exists

Core 2 already gives the deeper substrate:
- carriers
- sites
- branching and joining
- axis-based attitude
- frame-relative placement
- tension and activation
- boolean splitting over framed spans

What this layer adds is a reusable symbolic vocabulary for concepts such as:
- containment
- adjacency
- crossing
- source-goal
- branching
- closure
- ordering
- hierarchy
- temporal interval
- support
- attachment
- center-periphery
- obstruction

These are intentionally more abstract than any one language, but more structured than loose free-text descriptions.

## Surface Word Classes It Should Help With

The intent is not only to cover prepositions.
The same small family of schemas should be reusable across:
- prepositions and spatial relators
- temporal words such as `during`, `until`, and `throughout`
- motion and path verbs such as `cross`, `exit`, and `approach`
- dimensional and scalar adjectives such as `tall`, `deep`, and `very`
- kinship, social distance, and hierarchy terms such as `close friend` and `upper management`
- dynamic equation patterns such as iterative update, comparison, branching, and stopping bands

At the symbolic layer these are not separate ontologies.
They are different lexical surfaces over overlapping structural families.

## Working Principle

This layer should prefer:
- physically interpretable structure
- host-relative readings
- boolean and segment-based region logic where useful
- reuse across spatial, temporal, procedural, social, and narrative domains
- enough generality that the same schema can be reused in letters, circuits, graphs, and language
- surface forms that remain close to what Core 2 can already encode with carriers, sites, tensions, and frame-relative spans

It should avoid:
- arbitrary word-to-number mappings
- language-specific one-off enums where a reusable structural family already exists
- collapsing alternative readings too early

## Minimal Abstract Vocabulary

The current draft assumes a small recurring backbone:
- `Carrier`
  continuity, route, thread, traversal, stroke
- `Site`
  local encounter, attachment, branch, boundary, closure
- `Relation family`
  containment, adjacency, source-goal, crossing, hierarchy, and related schemas
- `Attitude`
  local axis-based preference, opposition, orthogonality, and weighting
- `Continuity`
  same stroke or thread versus a new one
- `Multiplicity`
  co-present versus alternative continuation
- `Tension`
  unresolved error, pressure, activation, or exploratory drive

The folding shape can be interpreted later.
This layer is mainly about preserving the reusable relation family before one display or enactment is chosen.

## Current Direction

The first pass is intentionally light:
- relation-family enums
- topology-kind enums
- lexical-domain enums
- a small schema draft record
- a catalog of reusable schema drafts that can be inspected from multiple domains
- a catalog of lexical or conceptual mapping examples with loose Core 2-shaped formulas

The applied language layer can then map English items onto these draft schemas and reveal where the abstraction is solid or where it is stretching too hard.
