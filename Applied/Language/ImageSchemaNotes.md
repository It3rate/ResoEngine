# Image Schema Notes

This file is a rough bridge to image-schema style thinking and localist cognitive semantics.

The point is not to commit to any one school of terminology.
The point is to preserve overlap with recurring structural patterns that Core 2 may be able to encode directly.

## Recurring Schemas

- `Container`
  inside, outside, entering, leaving
- `Path`
  from, to, along, across, through
- `Source-Path-Goal`
  movement, planning, causation, procedural progression
- `Link`
  attachment, connection, relation, circuit closure
- `Center-Periphery`
  center, edge, margin, core, fringe
- `Up-Down`
  vertical organization, hierarchy, intensity, control
- `Front-Back`
  ordering, visibility, priority, temporal before/after
- `Near-Far`
  adjacency, relevance, intimacy, uncertainty band
- `Branch-Merge`
  alternatives, control flow, circuit splits, discourse divergence
- `Closure-Loop`
  cycles, enclosure, return, recurrence
- `Blockage`
  obstacle, guard, tension, interrupted continuation
- `Part-Whole`
  member, portion, inside the group, belonging to a larger structure
- `Support-Cover`
  over, under, held by, carried by, overshadowed by

## Working Hypothesis

Letters and glyphs may be readable because they reuse many of these same primitive patterns:
- branch
- merge
- closure
- corner
- crossing
- return
- apex
- center-periphery
- baseline and topline anchoring

If that is true, then the path from Core 2 into language should not rely on arbitrary symbolic assignment.
It should rely on a small set of recurring schemas that remain intelligible across spatial, temporal, procedural, social, and graphical domains.

## Useful Cross-Domain Readings

| Schema | Spatial / prepositional reading | Temporal reading | Flow / graph reading | Social or scalar reading |
| --- | --- | --- | --- | --- |
| `Container` | inside, outside, within | during, within the week | inside a guarded region | within a group |
| `Path` | from, to, along | through the day, over a span | traversal or control path | progress toward a rank |
| `Boundary` | at the edge, beyond | until, by the cutoff | guard, threshold, event boundary | access barrier |
| `Link` | attached to, connected with | sustained dependence | graph edge, circuit tie | social tie |
| `Up-Down` | above, below | earlier-later when vertically framed | escalation / lowering stage | upper management, lower rank |
| `Near-Far` | near, beside, far | temporally close or distant | local versus remote node | close friend, distant relative |
| `Blockage` | blocked, obstructed | delayed until release | gated branch or interrupted path | gatekeeping or barrier |

## Encoding Intuition

The working intuition is that these should not become arbitrary symbolic tags.
Whenever possible they should be read as:
- carriers and spans
- sites and boundaries
- boolean interior or exterior regions
- source-goal routes
- branch or merge structure
- local axis-based attitude and weighting

So a later English lexicon should not replace the Core 2 substrate.
It should map words and constructions onto reusable structural families that already make sense in letters, traversal, circuits, and graphical organization.
