CORE 2 PROMPT AXIOMS
====================

This file is the living prompt-safe Core 2 specification.

Parts I through VIII are general Core 2 system axioms.
Part IX is a directed-segment appendix aligned to the current geometry and boolean implementation.
Part X integrates the current pinning redesign as host-relative attachment, encountered local structure, and later folding.


PART I. FOUNDATIONS
-------------------

F1. Fundamental duality.
The system is built from opposed pairs such as same/opposite, dominant/recessive, left/right, observer/opposite-observer, and encode/act.
Each such pair appears in two linked forms:
- an encoding that names a state or relation
- a transform that enacts that relation

F2. Encoding and action are paired but distinct.
Examples:
- 3 encodes triple scale; S_3 applies triple scale
- i encodes the opposite-unit state; J applies the opposite-unit action
- -1 encodes negation; N applies negation
The encoding and the transform are two formal appearances of the same underlying relation.

F3. Primitive object.
The primitive object is an ordered interval I = (p0, p1).
By itself it is not yet a number.

F4. No intrinsic measurement.
The interval has order but no intrinsic sign, size, or direction until it is placed in a frame.

F5. Frame.
A frame F provides:
- a zero point 0
- a positive-left unit uL
- a positive-right unit uR

F6. Perspective.
Perspective is binary in 1D:
- eps = +1 is dominant perspective
- eps = -1 is opposite perspective
Changing perspective changes the reading convention, not the interval itself.

F7. Endpoint-unit rule.
The first endpoint is always measured by the left unit.
The second endpoint is always measured by the right unit.
The interval's asymmetry comes from endpoint order plus unit attachment.

F8. Reading map.
A reading of the interval in frame F and perspective eps is:
rho(F, eps, I) = ai + b
where:
- a measures p0 in the left unit
- b measures p1 in the right unit
Here i is initially an encoding, not yet an operator.

F9. Observer conversion.
Opposite perspective negates the reading:
rho(F, -eps, I) = -rho(F, eps, I)
So:
ai + b -> -ai - b

F10. State and transform are asymmetric in multiplication.
In x * t:
- x is the current state or readout
- t is transform data
So i * i means "take state i, then apply J once."
It does not mean "there are two swaps in the expression."

F11. Scalar transform.
For real k, let S_k be the scale transform.
Then:
- S_1 = Id
- S_-1 = N
- S_a o S_b = S_(ab)
Any real scalar factor in operator position transforms the interval itself.

F12. Opposition transform.
J is the opposition transform.
It does not merely relabel a symbol.
It changes which unit is attached to each endpoint.
Multiplying by i in operator position means "apply J":
x * i := J(x)

F13. Canonical opposition cycle.
Under dominant notation:
- J(1) = i
- J(i) = -1
- J(-1) = -i
- J(-i) = 1
So J generates the four-state cycle:
1 -> i -> -1 -> -i -> 1

F14. Operator-level law.
At the transform level:
J o J = N
Therefore:
J^4 = Id

F15. Addition.
Addition is defined only after common frame and common perspective are fixed.
Then:
(ai + b) + (ci + d) = (a + c)i + (b + d)

F16. Dominant/recessive convention.
Dominant and recessive are not extra geometry added to the bare interval.
They are naming conventions under the dominant perspective, where dominant direction is encoded to the right.

F17. Derived products.
The familiar state cycle follows from the state/transform rule:
- 1 * i = i
- i * i = -1
- -1 * i = -i
- -i * i = 1
The familiar i^2 = -1 is therefore derived shorthand, not a primitive statement about two operators.

F18. Scalar and opposition can compose.
If 3i appears in transform position, it means "scale by 3 and apply opposition."
So one may write:
T_(3i) = S_3 o J
or
T_(3i) = J o S_3
depending on the chosen operator ordering convention.

F19. Core ontology.
A directed value is not fundamentally a number.
It is an ordered interval placed into a frame and then read from a perspective.

F20. Local description and later placement differ.
A structured object may carry an intrinsic local frame before later placement in another host frame.
Local description and embedded reading should not be collapsed prematurely.


PART II. REPETITION AND INVERSE CONTINUATION
--------------------------------------------

R1. Repetition is general.
Repetition is not limited to exponentiation.
A repetition is any indexed family generated by a continuation law.

R2. Every repetition has three parts.
Every repetition has:
- a carrier: what is being repeated
- an index: how repetition advances
- a continuation law: how one instance produces the next

R3. Carriers may vary.
A carrier may be:
- a value
- a transform
- a frame
- a pattern
- a relation
- a recursive state

R4. Indices may vary.
An index may be:
- discrete step
- continuous phase
- spatial displacement
- dimensional degree
- event count
- recursion depth

R5. Continuation law determines what "again" means.
Examples include:
- identical repeat
- translated repeat
- reflected repeat
- wrapped repeat
- alternating repeat
- scaled repeat
- damped repeat
- tension-preserving continuation

R6. Exponentiation is a special case.
Exponentiation is repeated transform application under a fixed continuation law.
Examples:
- repeated addition
- repeated multiplication

R7. Boundary behavior is repetition.
When a value reaches a frame boundary, its further behavior is determined by a continuation law.
Examples:
- wrap: periodic repetition
- reverse: reflective repetition
- oscillate: smooth alternation
- continue beyond the frame: tension-preserving continuation
- clamp: hold at the boundary
In the current pinning direction, these law names are compatibility summaries of encountered endpoint-pin behavior rather than the deepest source of continuation meaning.

R8. Patterns are repetitions.
Tilings, waves, alternating signs, recurring quadrant structures, and path motifs are repetitions of local rules across a space.

R9. Recursion is repetition.
A recursive system is a repetition whose next state is generated from earlier states rather than from one fixed transform alone.

R10. Repetition may preserve or transform.
A repetition may return the same motif exactly, or modify it on each return.

R11. Repetition acts on more than values.
Repetition may act on:
- values
- transforms
- frames
- operations
- recursive constructions
- display patterns

R12. Tension is a valid repetition outcome.
If repetition reaches a limit that is not resolved by the current frame, the system may preserve the result as tension rather than forcing a collapse.

R13. Inverse continuation.
An inverse continuation is the attempt to reverse a repetition.
Examples:
- roots as inverse of repeated multiplication
- logarithms as inverse of repeated exponentiation
- unwrapping as inverse of periodic continuation

R14. Inverse continuation is generally non-unique.
Forward repetition may map multiple inputs to the same result.
Therefore reverse continuation often yields multiple valid candidates.

R15. Candidate set first.
For an inverse continuation problem, the full set of valid candidates should be determined before selection unless the governing law yields only one candidate.

R16. Shared selection priority.
When choosing among inverse or branch candidates, the system should prefer:
- structural validity
- continuity from the prior state, if present
- preservation of degree-specific roles
- preservation of current perspective for display, if relevant
- minimal added tension

R17. Tension preservation in inversion.
If no unique inverse candidate is justified, ambiguity may remain as tension instead of being collapsed prematurely.

R18. Principal branch and all-candidates mode.
A principal branch is the default selected reverse continuation when one representative is required.
An all-candidates mode preserves every valid reverse continuation explicitly.

R19. Degree-sensitive inverse continuation.
At degree 0, inverse continuation often collapses to ordinary arithmetic ambiguity.
At degree 1, inverse continuation acts on dominant amount and recessive support.
At degree 2, inverse continuation acts on oriented values and may depend on perspective, orbit continuity, and role preservation.
At degree 3, inverse continuation splits between fold-first and structure-preserving modes.
At higher degrees, inverse continuation generally becomes more ambiguous because more structure can vary while preserving the same folded result.

R20. Preferred inverse-continuation behavior by degree.
For Proportion:
- preserve the full lawful candidate set, including negative or zero recessive support when generated
- allow principal display to prefer positive recessive support when a single representative is required
- do not silently move a recessive sign into the dominant side merely to relieve tension
- rise to a higher degree if proportion alone cannot support the continuation
For Axis:
- default to the continuation that best preserves the current oriented reading when no other rule is supplied
For Area:
- prefer structure-preserving inversion when the expanded structure is present
- accept fold-first inversion when only the folded result is available

R21. Inverse continuation is reverse path selection.
Inverse continuation is not merely the reversal of a number.
It is the selection of a valid reverse path through a repetition law.


PART III. STRUCTURE, QUANTITIES, AND UNITS
------------------------------------------

U1. Structure and units are separate.
Structural degree and unit-kind are distinct.
Examples:
- Axis, Area, and Volume describe structure
- distance, time, mass, and charge describe unit-kind

U2. A quantity has layered structure.
A quantity consists of:
- a structural element
- a unit signature
and may also carry:
- a chosen named unit
- a physical referent
- tensions

U3. Unit signatures form a free abelian group.
A unit signature is a product of abstract generators with integer exponents.

U4. Unit operations.
Unit-signature operations are:
- multiply: add exponents
- divide: subtract exponents
- reciprocal: negate exponents
- power: multiply exponents by the power

U5. Addition requires matching signature.
Two quantities may be added or subtracted only when their signatures match, unless the mismatch is intentionally preserved as tension.

U6. Named units are presentation choices.
Meters, feet, seconds, and similar named units are expression choices for a signature.
They do not change the signature itself.

U7. Scale and signature differ.
Two units may share a signature while differing in scale.

U8. Physical meaning may be attached later.
A unit generator may begin abstract and later be associated with a physical meaning.

U9. Referent and signature differ.
Different physical domains may share the same algebraic signature while differing in meaning, constraints, and interpretation.

U10. Physical reality constrains unit use.
Although unit algebra is abstract, it can carry:
- semantic meaning
- empirical limits
- allowed ranges
- compatibility rules

U11. Structural and unit algebras compose.
Operations on quantities act separately on:
- structure
- unit signature
and are reconciled afterward.

U12. Unit tension is informative.
Examples:
- signature mismatch
- scale mismatch
- unresolved referent
- incompatible physical meaning
- constraint violation

U13. Combined principle.
A mathematical object in this system may carry:
- structure
- interpretation
- repetition law
- unit signature
- physical referent
- tension

U14. System aim.
The framework distinguishes:
- what an object is
- how it repeats
- how it is inverted
- how it is measured
- what it means physically
- where unresolved information remains


PART IV. MULTISCALE RESOLUTION AND LAYERED READINGS
---------------------------------------------------

M1. Resolution frame.
A resolution frame defines a grain at which a quantity may be read.
The frame carries:
- a signature
- a grain
- an optional named unit or symbol

M2. Resolution ladder.
A resolution ladder is an ordered family of resolution frames sharing one signature and descending from coarser grain to finer grain.

M3. Layered quantity.
A layered quantity is an exact quantity represented as a sum of signed counts across one or more resolution frames.

M4. Signed digits are allowed.
Resolution counts need not be only nonnegative digits.
A layered quantity may use signed counts when that representation is structurally useful.

M5. Exact fold.
Folding a layered quantity means summing all frame-count contributions into one exact quantity in the shared signature.

M6. Exact decomposition.
A quantity may be decomposed across a resolution ladder into frame counts.
This creates a place-value-like representation at several grains.

M7. Coarse reading.
Reading at a chosen frame does not require discarding exact structure.
The system may compute:
- the raw tick count at that grain
- the quantized tick count
- the representative quantity at that grain
- the residual detail not captured by that reading

M8. Quantization rule.
A coarse reading depends on a quantization rule such as nearest, floor, ceiling, toward-zero, or away-from-zero.

M9. Fine detail may collapse to zero.
When a quantity is read in a coarse frame, contributions from finer frames may collapse to zero at that grain even though they remain present in the exact folded quantity.

M10. Refinement and collapse.
Refinement introduces or preserves finer-frame detail.
Collapse re-expresses the same quantity at a coarser frame, possibly with residual detail omitted from the representative reading.

M11. Representative and residual differ.
A representative reading is the quantity expressed at the chosen grain.
A residual is the remaining exact detail not captured by that representative.

M12. Current Core 2 scope.
The current implementation begins with scalar quantities paired with unit signatures.
Higher-structure multiscale readings may be added later.


PART V. GENERAL BRANCHING
-------------------------

B1. Branching is general.
A lawful process is not required to remain single-track.
Whenever a continuation, inversion, decomposition, or constraint system yields more than one valid outcome, the system forms a branch family.

B2. Linear path.
A linear path is a continuation in which each present state has at most one valid next state and at most one valid reverse continuation within the fixed frame, perspective, degree, and repetition law.

B3. Branch family.
For a law L and state x, the branch family B_L(x) is the full set of valid candidates generated by L acting on x, forward, reverse, or structurally.

B4. Branch metadata.
A branch family should retain:
- generating law
- direction of generation
- branch type or semantics
- candidate set
- tensions
- selection state
- annotations or notes

B5. Preimage branching.
Preimage branching arises when distinct states are sent by a law to the same displayed result, so reverse continuation yields several valid antecedents.

B6. Component branching.
Component branching arises when one structured object yields several connected components under an operation.
These branches are co-present, not mutually exclusive.

B7. Continuation branching.
Continuation branching arises when one present state admits several lawful next states.

B8. Extension branching.
Extension branching arises when the current frame, degree, or continuation law cannot carry all active distinctions on a single represented path.
The system may then lift to a richer representation rather than force collapse.

B9. Observer multiplicity is not structural branching by itself.
Different observer readings of the same structure are not, by themselves, structural branches.
Perspective changes display unless the system is explicitly branching over perspectives.

B10. Branch valence.
Branch valence is the number of immediate candidates in a branch family.

B11. Branch depth.
Branch depth is the number of successive branch events along a continuation.

B12. Lift.
A lift is an added coordinate, phase, sheet, index, time offset, or structural degree introduced so previously collapsed distinctions can be represented without loss.

B13. Lift principle.
When a valid distinction cannot be carried on the current linear path, the system may lift the path before forcing a selection.

B14. Rejoin law.
A rejoin law is a law under which distinct branches later map to a common state or displayed result.

B15. Branching and repetition.
Repetition does not itself require branching.
Branching appears when repetition is inverted, folded, wrapped, phase-shifted, or constrained so that several valid continuations or antecedents arise.

B16. Alternation.
Alternation is periodic continuation with phase.
It is not automatically a true split into independent branches.
It becomes branch-relevant when phase is hidden, when inverse continuation must recover phase, or when different phases are treated as separate candidates.

B17. Contradiction and tension.
If active requirements cannot be simultaneously realized in the current representation, the system should not force premature collapse.
It may preserve the contradiction as tension, lift to a richer representation, produce a branch family, or select a representative by rule if justified.

B18. Selection.
Selection is the act of choosing one candidate from a branch family when a single continuation is required for display, action, or further computation.
Selection should not erase the existence of the full family.

B19. Principal branch.
A principal branch is the default selected representative when one candidate must stand in for the whole family.

B20. All-candidates mode.
In all-candidates mode, the branch family is preserved explicitly and no candidate is erased merely because one is selected for display.

B21. Branch tension.
If no unique selection is justified, unresolved multiplicity may remain as branch tension.
Branch tension is informative structure, not failure.

B22. Alternative and co-present branches differ.
Alternative branches compete for the same role and may require selection.
Co-present branches jointly constitute the result and should not be collapsed unless a later law explicitly folds them.

B23. Computational principle.
A computational system requires explicit handling of:
- selection
- preservation of alternatives
- lifting
- rejoining
- tension
Branching therefore belongs to execution as well as description.


PART VI. BRANCH FAMILIES AND BRANCH GRAPHS
------------------------------------------

G1. Branch families are local multiplicity events.
A branch family records the valid multiplicity produced by one lawful event.
It does not by itself encode the whole long-range graph.

G2. Branch members carry provenance.
Each branch member may retain parent identifiers from the prior multiplicity event.
This allows later continuations to preserve how candidates were produced and where rejoining occurred.

G3. Projection preserves branch identity when possible.
When a branch family is mapped into another domain, member identity, parent links, selection state, and annotations should be preserved unless the law explicitly merges or discards them.

G4. Merging requires explicit rejoin.
If distinct parents yield the same projected result, the projected member may rejoin them into one value while retaining all contributing parent identifiers.

G5. Inverse continuation uses alternative reverse families.
Roots are the basic example.

G6. Directed segment boolean splitting uses co-present structural families.
The resulting pieces are jointly the result, not competing alternatives.

G7. Fractional powers may carry preimage provenance forward.
When a fractional power depends on inverse continuation, the later forward family may preserve the parent roots that generated each candidate.

G8. A branch graph is ordered history, not only multiplicity.
A branch graph records how branch families unfold under continuation across events.

G9. Frontier marks current activity.
The frontier is the set of currently active branch nodes after an event.
Selection may mark one frontier node without deleting the others.

G10. Events and nodes are distinct.
An event is a branching moment.
A node is a concrete member produced at some event.

G11. Edges preserve structural relation.
Parent-child edges record how later nodes arise from earlier ones.
Edge kinds distinguish continuation, split, rejoin, merge, and lift.

G12. Rejoin preserves provenance.
When several parents lead to one child, the graph preserves all parents even if display later shows one node.

G13. Selection may itself be a graph event.
Selection may update the active continuation without creating a new structural node.

G14. Crossing does not imply joining.
Two paths may cross in display without forming a branch node.
A join must be explicit in the graph.

G15. Standalone graph attachment is allowed.
When a result is displayed without full prior history, its graph may attach to an explicit source frontier for standalone use.


PART VII. DYNAMIC EXECUTION
---------------------------

D1. Dynamic context has two layers.
A dynamic context consists of:
- a state
- an environment
The environment is not identical to the state.
It is the framing or surrounding condition within which the state gains executable meaning.

D2. Strands generate proposals.
A dynamic strand does not directly determine the next state.
It generates one or more proposals for what might happen next.

D3. A proposal may be partial.
A proposal need not solve the whole step by itself.
It may contribute only part of the next action, transform, or computation.

D4. Resolution is required.
When several strands act in parallel, the system must resolve their proposals before committing the next context.

D5. Resolution may return a branch family.
Dynamic resolution is not limited to a single next state.
It may return:
- one committed next context
- several co-present next contexts
- several alternative next contexts
- preserved tension without commitment
- a lifted continuation

D6. Tension is executable information.
Dynamic tension may arise from:
- conflict between proposals
- conflict with the environment
- structural invalidity
- aesthetic pressure
- unresolved multiplicity
Tension is part of execution, not merely a report after the fact.

D7. Environment feeds back.
The committed result of one step may alter the environment, and the altered environment may change what later strands propose or what the resolver prefers.

D8. A dynamic trace records becoming.
A dynamic trace is the ordered history of proposal, tension, resolution, and commitment.

D9. The branch graph records structural multiplicity, not the dynamic law itself.
The branch graph is the structural history of how dynamic resolution split, continued, selected, or rejoined.

D10. Convergence is a lawful stop condition.
A dynamic system may stop by:
- fixed step bound
- empty frontier
- stable orbit
- repeated frontier
- sufficiently resolved tension
The stop condition is part of the dynamic law.


PART VIII. TRAVERSAL AND STATEFUL CARRIERS
------------------------------------------

T1. Traversal is stateful.
A traversal is not memoryless.
It retains current value, continuation law, and facing or phase between firings.

T2. Traversal definition.
A traversal definition may include:
- a frame, if bounded continuation is desired
- a step
- a continuation law
- a seed or initial value

T3. Seed is part of lawful state.
A nonzero seed is not merely display setup.
It changes the lawful path of later firings.

T4. Boundary law persists across firings.
Reflect, wrap, clamp, and tension-preserving continuation are not one-time adjustments.
They govern every later firing unless an explicit law-change occurs.

T5. Facing persists in reflective traversal.
Under reflective continuation, a traversal keeps its current facing until it reaches a boundary and reverses.
Later firings therefore depend on where and how earlier firings ended.

T6. Bounded and unbounded traversals differ.
A traversal may use its carrier as a frame, or it may continue without a local frame.
This is part of the carrier definition.

T7. Step scale is part of the carrier.
A traversal may advance by unit step, span step, or another lawful step size.
The amount advanced on each firing is part of the carrier.

T8. Equation identity is stateful.
Two firings of the same named equation need not produce the same result.
The equation name identifies the carrier law, but the actual outcome depends on current internal state.

T9. Offset carrier rule.
When a carrier begins away from zero, that offset belongs to the traversal itself.
It is not merely a one-time pre-roll before the real behavior begins.

T10. Hidden lead under path interpretation.
Under a path interpretation, the route from zero to the carrier start may be treated as a hidden lead.
A hidden lead repositions the cursor without drawing.

T11. Visible span under path interpretation.
Under the same interpretation, the route from carrier start to carrier end is the visible span unless another law says otherwise.

T12. Hidden lead repeats with the carrier.
If a carrier has a hidden lead, that hidden lead is part of every lawful firing.
It is not consumed after first use.

T13. Commit materializes visible accumulation.
A commit closes the currently visible accumulation into a path result, but it does not reset the internal state of the traversals that produced it.

T14. Multiple fires may share one commit.
Several equations may fire before one commit.
Their combined visible effect then constitutes one grouped step or tick.

T15. Stateful carriers make repetition executable.
The same carrier may reflect, wrap, continue, delay, or leave gaps without changing identity, because its current state is part of what the carrier is.

T16. Traversal may encounter pins before boundaries.
A traversal is not limited to reacting only at the outer frame.
Interior landmarks and endpoint pins may be encountered before a boundary is exceeded, and those encountered pins may absorb, redirect, branch, or reframe the continuation.

T17. Boundary-law labels are traversal summaries.
Named boundary laws such as reflect, wrap, clamp, and tension-preserving continuation remain useful summaries for display and authoring.
They should increasingly be understood as shorthand for encountered endpoint-pin configurations rather than as the primitive source of traversal behavior.

T18. Encountered pin behavior is host-relative.
When traversal encounters a placed operand on a carrier, meanings such as same-carrier and orthogonal-carrier are read relative to the host carrier of that encounter rather than by absolute carrier labels.

T19. Approach and continuation may differ.
An encountered placed operand may support or oppose the approach side and may independently support or block continuation beyond the encounter.
Traversal should preserve that distinction rather than flattening the whole event into one reaction.


PART IX. DIRECTED SEGMENT BOOLEAN RULES
---------------------------------------

S1. Boolean segment results require a frame.
Boolean occupancy on directed segments is evaluated inside a common frame.
If no explicit frame is supplied, the system uses the convex envelope of the compared segments.

S2. Complement is frame-relative.
NOT A is not defined as a standalone segment transform.
It means the complement of A within the active frame.

S3. Boolean results are piece families.
A boolean segment result is not required to collapse to a single segment.
It is a co-present family of directed pieces generated by the truth pattern across the frame.

S4. Truth is evaluated on constant partitions.
The frame is partitioned by all relevant boundaries.
Within each partition, truth is constant, so each partition can be evaluated once and either kept or discarded.

S5. Direction carrier rule.
Each surviving boolean piece inherits direction and support from a carrier:
- if only A occupies the partition, use A
- if only B occupies the partition, use B
- if both occupy the partition, use dominant A
- if neither occupies the partition but the boolean rule is still true, use the frame

S6. Frame carrier rule.
When truth is produced only by negation or exclusion, the frame carries the direction of the resulting piece.

S7. Compatibility-preserving merge rule.
Adjacent true partitions may merge into one piece only when their carriers are structurally compatible.
Compatibility requires matching direction, basis, and supports.
Otherwise the result remains segmented.

S8. Envelope is not boolean union.
The convex envelope of two directed segments is not the same as boolean OR.
Envelope fills gaps; boolean OR preserves them.

S9. Primary-direction intersection rule.
When a single overlap segment is requested, the overlap is expressed in the direction and support frame of the primary segment.

S10. Co-present multiplicity must be preserved.
When boolean evaluation yields multiple surviving pieces, those pieces jointly constitute the result and should not be collapsed unless a later law explicitly folds them.


PART X. PINNING, PLACEMENT, AND FOLDING
---------------------------------------

P1. Pinning is fundamental.
Pinning is not merely a side feature of units or a display convenience.
It is a foundational local mechanism by which structure is attached, placed, redirected, interrupted, lifted, folded, branched, or rejoined.

P2. Pinning and folding differ.
Pinning creates structural relation.
Folding is a later interpretation or enactment of that relation.
A pinned structure may remain lawful and informative before any one folded result is chosen.

P3. One pin mechanism is preferred.
There should not be many primitive named pin types.
Wrap, reflect, continue, clamp, branch, corner, loop, and join are best understood as contextual readings of one pinning mechanism.

P4. Pinning remains locally binary.
A point pinning event joins two structural roles at one site:
- a host role
- an applied role
Richer structures arise by nesting, sequencing, or distributing such events rather than by introducing many-way primitive pins.

P5. A * B @ P is the canonical positioned pinning form.
The expression A * B @ P should be read first as a pinning event:
- A is the host
- B is the applied element
- P is the host-relative pin position

P6. Host is canonically recessive.
The host role is canonically recessive.
It supplies the local substrate, carrier, and attachment site of the event.

P7. Applied is canonically dominant.
The applied role is canonically dominant.
It supplies the local directive brought to the host at the pinning site.

P8. Pinning aligns dominant with dominant.
When host and applied are pinned, dominant direction is aligned with dominant direction at the attachment site.
Negative and orthogonal consequences are then read relative to that alignment.

P9. Intrinsic local frames belong to the operands.
Both host and applied operands may already be straight, bent, inverted, or unresolved before pinning.
Those properties belong to the operands themselves, not to the pin as an added bend-generator.

P10. Axis is the current canonical intrinsic descriptor.
The existing Axis remains the current canonical descriptor carried by a pinned operand.
It need not be treated as a separate extra object owned by the pin.

P11. The four descriptor slots remain distinct.
The current descriptor carries:
- a recessive unit
- a recessive value
- a dominant unit
- a dominant value
In shorthand these may be read as i, iV, u, and uV.

P12. Unit and value must remain distinct.
The unit slots describe carrier preference, contradiction, or unresolved carrier assignment.
The value slots describe realized extent, travel, or enactment on the chosen carrier.
Taken together they express local structural desire, not guaranteed final realization.

P13. Unit sign gives coarse structural type.
For a pin-side descriptor:
- positive unit prefers the current carrier
- negative unit prefers orthogonal breakout relative to the current carrier
- zero unit preserves unresolved carrier choice
Negative unit does not mean globally vertical by itself.
It means orthogonal relative to the hosted carrier being read.

P14. Value sign acts within the chosen carrier.
Once a carrier has been preserved, lifted, or otherwise selected:
- positive value realizes with that carrier's orientation
- negative value realizes against it
- zero value preserves unrealized travel

P15. Magnitude is weighted participation.
Values beyond -1, 0, and 1 should not be treated as meaningless extras.
Their magnitude is best read as weighted participation, strength, or local transport pressure rather than as a different primitive kind of pin.

P16. Pins require location.
A pinning event must be locatable in some frame, carrier, sheet, or relation.
Location is part of what makes the event interpretable.

P17. Location and attachment differ.
A pin may be locatable without yet being structurally attached.
Attachment is join; location is situatedness.

P18. Line-like hosts use proportional pin position.
For line-like hosts, the pin position P is normally represented by Proportion.
It is measured in the host's frame and support rather than in a detached neutral frame.

P19. Local coordinates and host embedding differ.
The host-relative position of a pin, the local zero of the applied element, and the later embedded reading are distinct layers.
They should not be collapsed prematurely.

P20. Point pinning and distributed pinning are one mechanism at different extents.
Point pinning is one local attachment event.
Distributed pinning is a family of such local events enacted across a support, span, or piece family.

P21. Distributed pinning belongs to folding.
Point pinning gives local structural potential.
Distributed pinning is often the enacted form of a later fold, such as support-wide comparison, continuation, span formation, or similar combination.

P22. Opposed same-carrier resolution yields directed-segment structure.
If the resolved sides occupy one carrier and point against each other, the local result is directed-segment structure.

P23. Same-direction same-carrier resolution yields reinforcement.
If the resolved sides occupy one carrier and point in the same direction, the local result is reinforcement, acceleration-like continuation, or layered same-direction passage.

P24. Orthogonal resolution yields corner-like or area-like structure.
If the resolved sides occupy perpendicular carriers, the local result is corner-like, branch-like, or area-like structure.

P25. Zero or partial resolution is lawful.
One side may resolve while the other remains noisy, latent, or unresolved.
If both sides remain unresolved, the pin may remain as held tension or ambiguous potential rather than forcing false structure.

P26. Preferred geometric breakout is the smallest lawful orthogonal lift.
When a carrier contradiction is resolved geometrically, the preferred first response is the smallest lawful orthogonal lift available at the current structural grade.

P27. Meaning depends on location.
The same applied descriptor may read differently at:
- an endpoint
- an interior landmark
- an off-carrier location
- a relation between carriers
- a relation between sheets

P28. Open endpoint is not clamp.
If a carrier reaches its end with no explicit resolving pin, this is not a stop.
It is open termination, and it raises tension.

P29. Endpoint law names are contextual readings.
Clamp, reflection, continuation, wrap, and loop are best understood as contextual readings of encountered endpoint or landmark structure rather than as the deepest primitive laws.

P30. Pins may attach, emit, or seek attachment.
A pin may attach to an existing carrier, emit a new carrier, or do both.
Growth and join are therefore not fully separate.

P31. A pin may have zero, one, or two local outputs.
Locally, a pin may:
- absorb with zero outputs
- continue or redirect with one output
- branch with two outputs
Richer multiplicity should generally be represented by several pins or a branch family.

P32. Sheets differ from observer perspective.
The places in which pins and carriers interact should be distinguished from the binary observer +/- perspective.
Sheets or fields are the better layer for multi-place interaction.

P33. Host-relative carrier meaning must be preserved.
Carrier interpretations such as same-carrier and orthogonal-carrier are relative to the host carrier of the encounter.
They should not be treated as permanent absolute labels such as "carrier 0" and "carrier 1".

P34. Boundary labels remain compatibility shells.
Named boundary laws remain useful for display, editing, and compatibility.
They should increasingly be understood as shorthand for explicit endpoint-pin configurations.

P35. Boundary-relative pins preserve intent under reframing.
When a boundary-relative pin is moved onto a new carrier or route frame, its left/right intent should be preserved if that meaning can be translated lawfully onto the new boundaries.

P36. Tension must not be normalized away prematurely.
Negative or zero unit information, ambiguous attachment, unresolved continuation, or competing pin goals should not be silently normalized away merely to recover a simpler display.
Such tension may later drive rupture, branching, curvature, phase offset, compromise, or higher-dimensional lift.

P37. Pinned structure and folded result should preserve provenance.
Whenever possible, a folded result should preserve:
- where the pin occurred
- which operand acted as host
- which operand acted as applied
- which intrinsic descriptors were involved
- what unresolved tensions remained


KEY EXAMPLES
------------

E1. Core opposition cycle.
1 * i = i
i * i = -1
-1 * i = -i
-i * i = 1

E2. Observer flip.
If one observer reads an interval as 2i + 3, the opposite observer reads it as -2i - 3.
The interval is the same; only the perspective changed.

E3. Inverse continuation.
x^2 = 4 yields the reverse candidates 2 and -2.

E4. Component branching.
Removing a middle segment from a longer segment yields two co-present remaining pieces.

E5. Directed segment boolean split.
If A = [0, 10] and B = [3, 5], then A XOR B yields [0, 3] and [5, 10].

E6. Reflective carrier.
A reflective carrier on [0, 1] with unit step alternates direction on every firing.
A reflective carrier on [0, 2] with unit step advances across two ticks before reversing.

E7. Offset carrier under path interpretation.
A carrier on [-1, 1] may first reposition silently to -1 and then draw to 1.
On later firings, that silent-plus-visible structure remains part of the same equation.

E8. Multiscale decomposition.
109 miles may be decomposed as 1 x 100-mile + 9 x 1-mile.

E9. Signed digits.
The signed-digit form 1, 0, -2 in base 10 folds to 98.

E10. Coarse reading.
Reading 101 miles at the 100-mile frame may yield the representative 100 miles with residual 1 mile.

E11. Pinning default.
A positive recessive unit and positive dominant unit produce the current straight directed-segment unit.

E12. Pinning contradiction.
A negative pin-side unit may contradict its native line-carrier and prefer orthogonal lift rather than silent sign normalization.

E13. Positioned pinning.
A * B @ 1/2 means:
- pin B onto A
- at the midpoint of A
- by aligning dominant direction with dominant direction
- then interpret the resulting placed structure

E14. Distributed pinning as fold.
A support-wide comparison or multiplication-like construction may be read as a later fold that enacts many local pinning relations across a shared support rather than as one isolated point event.

E15. Same-direction reinforcement.
Two resolved pin sides pointing in the same direction may act like sequential reinforcement or acceleration.

E16. Zero-unit hold.
A zero unit may preserve unresolved or noisy carrier information without committing to one orientation.

E17. Open endpoint.
A carrier that reaches an end without an explicit resolving pin does not thereby clamp.
It produces open termination tension.

E18. Explicit endpoint pin.
A clamp, reflect, or wrap boundary may be represented by explicit endpoint pins rather than only by a named continuation-law enum.

E19. Interior landmark pin.
A traversal may encounter an interior landmark before the frame edge and reverse, absorb, or jump from that landmark without changing the identity of the carrier that encountered it.


KEY GLOSSARY
------------

Branch family:
The full set of valid outcomes produced by one lawful event.

Alternative branch:
One of several candidates competing for the same role.

Co-present branch:
One of several pieces that jointly constitute the result.

Branch tension:
Unresolved multiplicity preserved without premature collapse.

Lift:
An added coordinate, phase, sheet, translation, time offset, or structural degree used to carry otherwise-collapsed distinctions.

Rejoin law:
A law under which distinct branches later meet again in one displayed result.

Frontier:
The set of currently active branch nodes after an event.

Dynamic context:
A paired state and environment used for execution.

Traversal:
A stateful carrier driven by a step, a continuation law, and possibly a frame and seed.

Hidden lead:
Under path interpretation, the non-drawing portion of a carrier that repositions the cursor before the visible span.

Resolution frame:
A grain-bearing reading frame for a shared unit signature.

Resolution ladder:
An ordered family of coarser-to-finer resolution frames.

Layered quantity:
An exact quantity represented across more than one resolution frame.

Representative reading:
The quantity reported at the active grain after quantization.

Residual:
The exact detail not captured by the representative reading at that grain.

Pin descriptor:
An Axis read as a four-slot intrinsic descriptor carried into a pinning event.

Carrier contradiction:
Unit-level information that conflicts with the native carrier assignment of a pin side.

Orthogonal lift:
The preferred smallest geometric breakout used when a carrier contradiction cannot remain on the current carrier.

Open termination:
An unresolved endpoint encounter with no explicit resolving pin; it raises tension rather than acting as clamp.

Located pin:
A pin interpreted together with its location in a frame, carrier, sheet, or relation.

Endpoint pin:
A located pin encountered at a boundary or end of a carrier.

Landmark pin:
A located pin encountered at an interior or off-carrier landmark whose behavior is determined by descriptor plus context.
