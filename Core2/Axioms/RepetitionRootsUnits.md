**Repetition, Inverse Continuation, and Units**

1. **Repetition Is General** Repetition is not limited to exponentiation. A repetition is any indexed family generated by a continuation law.  
     
2. **Three Parts Of Repetition** Every repetition has:  
     
- a **carrier**: what is being repeated  
- an **index**: how repetition advances  
- a **continuation law**: how one instance produces the next  
3. **Possible Carriers** A carrier may be:  
- a value  
- a transform  
- a frame  
- a pattern  
- a relation  
- a recursive state  
4. **Possible Indices** An index may be:  
- discrete step  
- continuous phase  
- spatial displacement  
- dimensional degree  
- event count  
- recursion depth  
5. **Continuation Law** The continuation law determines what “again” means. Examples:  
- identical repeat  
- translated repeat  
- reflected repeat  
- wrapped repeat  
- alternating repeat  
- scaled repeat  
- damped repeat  
- tension-preserving continuation  
6. **Exponentiation Is A Special Case** Exponentiation is a repeated transform application under a fixed continuation law. Examples:  
- additive exponent: repeated addition  
- multiplicative exponent: repeated multiplication  
7. **Boundary Behavior Is Repetition** When a value reaches the boundary of a frame, its further behavior is determined by a repetition law. Examples:  
- wrap: periodic repetition  
- reverse: reflective repetition  
- oscillate: smooth alternating repetition  
- continue past the frame: tension-preserving repetition  
    
8. **Patterns Are Repetitions** Tilings, waves, alternating signs, and recurring quadrant structures are repetitions of local rules across a space.  
     
9. **Recursion Is Repetition** A recursive system is a repetition whose next state is generated from earlier states. Example: Fibonacci repetition draws new material from prior outputs instead of only repeating a single transform.  
     
10. **Repetition May Preserve Or Transform** A repetition may preserve a motif exactly, or transform it on each return. Examples:  
      
- same  
- opposite  
- mirrored  
- shifted  
- scaled  
- damped  
- phase-shifted  
11. **Repetition Acts On More Than Values** Repetition may apply to:  
- values  
- transforms  
- frames  
- operations  
- recursive constructions  
- display patterns  
    
12. **Tension Is A Valid Repetition Outcome** If repetition reaches a limit that is not resolved by the current frame, the system may preserve the result as tension rather than forcing a collapse.  
      
13. **Inverse Continuation** An inverse continuation is the attempt to reverse a repetition. Examples:  
      
- root as inverse of repeated multiplication  
- logarithm as inverse of repeated exponentiation  
- unwrapping as inverse of periodic continuation  
    
14. **Non-Uniqueness** Inverse continuation is generally not unique because forward repetition may map multiple inputs to the same result.  
      
15. **Branch Rule** A branch rule is the rule that selects one valid inverse continuation from among multiple candidates. In this system, “branch rule” and “inverse continuation rule” refer to the same idea.  
      
16. **Candidate Set** For an inverse continuation problem, the system should first determine the full set of valid candidates before selecting one. Selection should not erase the existence of alternatives unless explicitly required.  
      
17. **Selection Priority** When choosing a branch, the system should prefer:  
      
- structural validity  
- continuity from the previous state, if present  
- preservation of current degree-specific roles  
- preservation of current perspective for display  
- minimal added tension  
    
18. **Tension Preservation In Inversion** If no unique candidate is justified, ambiguity may be preserved as tension rather than collapsed prematurely.  
      
19. **Principal Branch** A principal branch is the default selected inverse continuation when a single representative is required.  
      
20. **All-Candidates Mode** The system may also operate in an all-candidates mode, where every valid inverse continuation is preserved.  
      
21. **Scalar Inverse Continuation** At degree 0, inverse continuation is minimal and often collapses immediately to ordinary arithmetic ambiguity. Example: `x^2 = 4` admits `2` and `-2`.  
      
22. **Proportion Inverse Continuation** At degree 1, inverse continuation acts on dominant amount and recessive support. A preferred rule is:  
      
- preserve positive recessive support when possible  
- allow dominant sign to carry the variation  
- rise to a higher degree if no valid continuation exists within proportion alone  
23. **Axis Inverse Continuation** At degree 2, inverse continuation acts on oriented values. Selection may depend on:  
- perspective  
- continuity of the current orbit  
- dominant/recessive role preservation  
- minimal transform displacement  
    
24. **Axis Principal Rule** For `Axis`, the principal branch should default to the continuation that best preserves the current oriented reading when no other rule is supplied.  
      
25. **Area Inverse Continuation** At degree 3, inverse continuation splits into two modes:  
      
- `fold-first`: invert only the folded 1D result  
- `structure-preserving`: invert the expanded 2D structure itself  
    
26. **Area Preference** When the expanded structure is present, structure-preserving inverse continuation is preferred. When only the folded result is available, fold-first inverse continuation is acceptable.  
      
27. **Higher Degrees** As degree increases, inverse continuation generally becomes more ambiguous because more internal structure can vary while preserving the same folded result.  
      
28. **Inverse Continuation As Reverse Repetition** Every inverse continuation should be understood as the reverse of some repetition law.  
      
29. **Branch Rules Depend On Repetition Laws** Different repetition laws create different inverse continuation problems. Examples:  
      
- repeated multiplication creates roots  
- periodic wrap creates unwrapping ambiguity  
- reflective continuation creates mirrored inverse paths  
30. **Display vs Structure** Branch selection should distinguish between:  
- the neutral structural candidate  
- the displayed reading of that candidate under a perspective  
    
31. **Perspective And Branch** Perspective may influence which inverse continuation is displayed as principal, but should not erase the existence of other structurally valid candidates.  
      
32. **Final Inverse Principle** Inverse continuation is not merely the reversal of a number. It is the selection of a valid reverse path through a repetition law.  
      
33. **Structure And Units Are Separate** The structural degree of a value and the unit-kind of a value are distinct. Examples:  
      
- `Axis`, `Area`, and `Volume` describe structure  
- distance, time, mass, and charge describe unit-kind  
34. **A Quantity Has Two Layers** A quantity consists of:  
- a structural element  
- a unit signature and may also carry:  
- a chosen named unit  
- a physical referent  
- tensions  
35. **Unit Signatures Form A Free Abelian Group** A unit signature is a product of abstract generators with integer exponents. Examples:  
- `L`  
- `T`  
- `L T^-1`  
- `L T^-2`  
36. **Unit Operations** Unit-signature operations are:  
- multiply: add exponents  
- divide: subtract exponents  
- reciprocal: negate exponents  
- power: multiply exponents by the power  
    
37. **Addition Requires Matching Signature** Two quantities may be added or subtracted only when their signatures match, unless the mismatch is intentionally preserved as tension.  
      
38. **Named Units Are Presentation Choices** Named units such as meters, feet, and seconds are choices of expression for a signature. They do not change the signature itself.  
      
39. **Scale And Signature Differ** Two units may share a signature while differing in scale. Example:  
      
- meter and foot share the same length signature  
- they differ in canonical scale  
40. **Physical Meaning Is Attached Later** A unit generator may begin as abstract and later be associated with physical meaning. Examples:  
- unknown generator `u0`  
- later identified as distance  
    
41. **Referent And Signature Differ** A physical referent is not the same thing as a signature. Different physical domains may share the same algebraic signature while differing in meaning and constraints.  
      
42. **Physical Reality Constrains Units** Although unit algebra is abstract, it ultimately maps back to physical reality. Physical referents may therefore carry:  
      
- semantic meaning  
- empirical limits  
- allowed ranges  
- compatibility rules  
43. **Structural And Unit Algebras Compose** Operations on quantities act separately on:  
- structure  
- unit signature and are reconciled afterward.  
44. **Tension In Units Is Informative** Examples of unit tension:  
- signature mismatch  
- scale mismatch  
- unresolved referent  
- incompatible physical meaning  
- constraint violation  
45. **Combined Principle** A mathematical object in this system may therefore carry:  
- structure  
- interpretation  
- repetition law  
- unit signature  
- physical referent  
- tension  
46. **System Aim** This framework distinguishes:  
- what an object is  
- how it repeats  
- how it is inverted  
- how it is measured  
- what it means physically  
- and where unresolved information remains

## Appendix: Branching, Lifts, Rejoining, and Selection

47. **Branching Is General**  
    A lawful process is not required to remain single-track. Whenever a continuation, inversion, decomposition, or constraint system yields more than one valid outcome, the system forms a branch family.  
      
48. **Linear Path**  
    A linear path is a continuation in which each present state has at most one valid next state and at most one valid reverse continuation within the fixed frame, perspective, degree, and repetition law.  
      
49. **Branch Family**  
    For a law `L` and state `x`, the branch family `B_L(x)` is the full set of valid candidates generated by `L` acting on `x`, forward, reverse, or structurally. If `B_L(x)` has one member, the continuation is linear. If it has more than one member, branching is present.  
      
50. **Branch Metadata**  
    A branch family should retain its generating law, direction of generation, branch type, candidate set, tensions, and any selected representative.  
      
51. **Preimage Branching**  
    Preimage branching arises when distinct states are sent by a law to the same displayed result. Reverse continuation then yields several valid antecedents. Roots, logarithms, modular unwrapping, and unfolded folded-results are examples.  
      
52. **Component Branching**  
    Component branching arises when one structured object yields several connected components under an operation. Segment subtraction that leaves two remaining pieces is an example. In component branching the branches are co-present, not mutually exclusive.  
      
53. **Continuation Branching**  
    Continuation branching arises when one present state admits several lawful next states. Choice points, control-flow branches, and multi-expansion recursions are examples.  
      
54. **Extension Branching**  
    Extension branching arises when the current frame, degree, or continuation law cannot carry all active distinctions on a single represented path. The system may then lift to a richer representation rather than force collapse. In the lifted representation, the apparent contradiction may become lawful as phase, translation, sheet, added degree, or preserved tension.  
      
55. **Observer Multiplicity**  
    Different observer readings of the same structure are not, by themselves, structural branches. Perspective changes display unless the system is explicitly branching over perspectives.  
      
56. **Branch Valence**  
    The branch valence is the number of immediate candidates in a branch family.  
      
57. **Branch Depth**  
    Branch depth is the number of successive branch events along a continuation.  
      
58. **Lift**  
    A lift is an added coordinate, phase, sheet, index, time offset, or structural degree introduced so previously collapsed distinctions can be represented without loss.  
      
59. **Lift Principle**  
    When a valid distinction cannot be carried on the current linear path, the system may lift the path before forcing a selection.  
      
60. **Rejoin Law**  
    A rejoin law is a law under which distinct branches later map to a common state or displayed result. Squaring rejoins `+/-` roots, modulo rejoins distinct lifts, and control flow may rejoin after a conditional.  
      
61. **Branching and Repetition**  
    Repetition does not itself require branching. Branching appears when repetition is inverted, folded, wrapped, phase-shifted, or constrained so that several valid continuations or antecedents arise.  
      
62. **Alternation**  
    Alternation is periodic continuation with phase. It is not automatically a true split into independent branches. It becomes branch-relevant when phase is hidden, when inverse continuation must recover phase, or when different phases are treated as separate candidates.  
      
63. **Contradiction and Tension**  
    If active requirements cannot be simultaneously realized in the current representation, the system shall not force premature collapse. It may preserve the contradiction as tension, lift to a richer representation, produce a branch family, or select a representative by rule if justified.  
      
64. **Selection**  
    A selection is the act of choosing one candidate from a branch family when a single continuation is required for display, action, or further computation.  
      
65. **Selection Preconditions**  
    Selection should occur only after the full candidate set has been determined, unless the governing law itself generates only one valid candidate.  
      
66. **Selection Priority**  
    When a single branch must be chosen, the system should prefer structural validity, continuity from the prior state if present, preservation of degree-specific roles, preservation of perspective for display if relevant, and minimal added tension.  
      
67. **Principal Branch**  
    A principal branch is the default selected representative of a branch family when one candidate must stand in for the whole family.  
      
68. **All-Candidates Mode**  
    In all-candidates mode, the branch family is preserved explicitly and no candidate is erased merely because one is selected for display.  
      
69. **Branch Tension**  
    If no unique selection is justified, unresolved multiplicity may remain as branch tension. Branch tension is informative structure, not failure.  
      
70. **Alternative and Co-Present Branches**  
    Alternative branches are candidates occupying the same role, so selection may be required. Co-present branches jointly constitute the result and should not be reduced to a single representative unless the law explicitly folds them.  
      
71. **Computational Principle**  
    A computational system requires not only values and transforms, but also explicit handling of selection, preservation of alternatives, lifting, rejoining, and tension. Branching therefore belongs to execution as well as description.  
      
72. **Final Branch Principle**  
    Branching is lawful multiplicity arising when distinctions exceed the carrying capacity of a single represented path. Such multiplicity may appear as alternatives, components, lifted phases, or reverse candidates. The system should preserve its origin and structure before collapsing it to a single display.

## Quick Glossary

- **Branch family**: the full set of valid outcomes produced by a law at some point.  
- **Lift**: an added way of carrying distinction, such as phase, sheet, translation, time offset, or extra structural degree.  
- **Rejoin law**: a law that causes different branches to meet again in one displayed result.  
- **Branch valence**: how many immediate candidates are available.  
- **Branch tension**: unresolved multiplicity preserved without premature collapse.  
- **Alternative branch**: one of several candidates competing for the same role.  
- **Co-present branch**: one of several pieces that jointly make up the result.

## Quick Examples

- **Preimage branching**: `x^2 = 4` gives candidates `2` and `-2`.  
- **Component branching**: subtracting a middle segment from a longer segment leaves two remaining segments.  
- **Continuation branching**: a recursive rule or control instruction allows either a left path or a right path next.  
- **Extension branching**: a constraint cannot be satisfied on one line, so the system lifts into phase, shift, glide, or a higher degree.  
- **Alternation**: `1, -1, 1, -1, ...` is a two-phase continuation; it is branch-relevant if the phase is hidden and later must be recovered.  
- **Rejoining**: `2` and `-2` rejoin under squaring because both map to `4`.
