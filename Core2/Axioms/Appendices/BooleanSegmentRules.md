# Directed Segment Boolean Rules

These are the additional rules implied by the corrected Core 2 boolean segment implementation.

## Proposed Appendix Rules

1. **Boolean Segment Results Require A Frame**  
   Boolean occupancy on directed segments is evaluated inside a common frame.  
   If no explicit frame is supplied, the system uses the convex envelope of the compared segments.

2. **Complement Is Frame-Relative**  
   `NOT A` is not defined as a standalone segment transform.  
   It means the complement of `A` within the active frame.

3. **Boolean Results Are Piece Families**  
   A boolean segment result is not required to collapse to a single segment.  
   It is a co-present family of directed pieces generated by the truth pattern across the frame.

4. **Truth Is Evaluated On Constant Partitions**  
   The frame is partitioned by all relevant boundaries.  
   Within each partition, truth is constant, so each partition can be evaluated once and either kept or discarded.

5. **Direction Carrier Rule**  
   Each surviving boolean piece inherits its direction and support from a carrier:
   - if only `A` occupies the partition, use `A`
   - if only `B` occupies the partition, use `B`
   - if both occupy the partition, use dominant `A`
   - if neither occupies the partition but the boolean rule is still true, use the frame

6. **Frame Carrier Rule**  
   When truth is produced only by negation or exclusion, the frame carries the direction of the resulting piece.

7. **Compatibility-Preserving Merge Rule**  
   Adjacent true partitions may merge into one piece only when their carriers are structurally compatible.  
   Compatibility requires matching direction, basis, and supports.  
   Otherwise the result remains segmented.

8. **Envelope Is Not Boolean Union**  
   The convex envelope of two directed segments is not the same as boolean `OR`.  
   Envelope fills gaps; boolean `OR` preserves them.

9. **Primary-Direction Intersection Rule**  
   When a single overlap segment is requested, the overlap is expressed in the direction and support frame of the primary segment.

10. **Co-Present Multiplicity Must Be Preserved**  
    When boolean evaluation yields multiple surviving pieces, those pieces jointly constitute the result and should not be collapsed unless a later law explicitly folds them.

## Quick Examples

- `A = [0, 10]`, `B = [3, 5]`, `A XOR B` yields two pieces: `[0, 3]` and `[5, 10]`.
- `NOT A` with frame `[0, 10]` and `A = [3, 5]` yields `[0, 3]` and `[5, 10]`.
- Two disjoint segments under boolean `OR` remain two pieces. Their envelope is a different result.
- A gap produced by `NOR` inherits the frame direction, not the direction of an absent segment.
