# Core 2 Dynamic

This folder is reserved for the dynamic execution layer of Core 2.

At the axiomatic level, dynamic execution is where:

- strands propose
- environments constrain
- tensions accumulate
- resolutions commit
- traces record becoming over time

## Current Status

This folder is intentionally light right now. The design direction is clearer than the implementation surface.

That is normal at this stage. A lot of dynamic behavior is still being exercised through symbolic programs, branching, repetition, and page-level experiments before it is consolidated here.

## Mental Model

Think of this folder as the future home of:

- executable proposal/resolution systems
- state plus environment dynamics
- multi-strand stepping
- dynamic traces and convergence logic

If `Branching` describes multiplicity structurally, `Dynamic` is where that multiplicity will eventually become executable process.
